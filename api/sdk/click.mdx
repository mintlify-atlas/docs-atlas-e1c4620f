---
title: "POST /api/sdk/click"
description: "Record that a user clicked on an ad"
---

Creates a click record for the specified ad. Call this endpoint when the user taps the ad's call-to-action button so that click-through rates and engagement data are captured in the dashboard.

## Authentication

This endpoint requires a valid API key passed in the `x-api-key` header.

## Request

### Headers

<ParamField header="x-api-key" type="string" required>
  Your application's API key.
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`.
</ParamField>

### Body

<ParamField body="adId" type="number" required>
  The numeric ID of the ad that was clicked, as returned by `GET /api/sdk/ads`.
</ParamField>

<ParamField body="deviceId" type="string">
  The unique identifier for the device. Optional but recommended for accurate per-device analytics.
</ParamField>

## Response

<ResponseField name="status" type="string">
  Returns `"ok"` when the click has been recorded successfully.
</ResponseField>

## Example Request

```bash
curl -X POST https://your-server.com/api/sdk/click \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"adId": 7, "deviceId": "a1b2c3d4e5f6"}'
```

## Example Response

```json
{
  "status": "ok"
}
```
